3. Logically Equivalent

public class Book {
    int id;
    String name;

    public Book(int id, String name) {
        this.id = id;
        this.name = name;
    }

    public boolean equals(Object obj) { //line n1
        boolean output = false;
        Book b = (Book) obj;
        if (this.name.equals(b.name)) {
            output = true;
        }
        return output;
    }

    public static void main(String[] args) {
        Book javaPg1 = new Book(101, "Java PG");
        Book javaPg2 = new Book(102, "Java PG");
        System.out.println(javaPg1.equals(javaPg2)); // line n2
    }

Which statement is true? A. The program prints true. B. The program prints false. C. A compilation error occurs. To ensure successful compilation, replace line n1 with: boolean equals (Book obj) { D. A compilation error occurs. To ensure successful compilation, replace line n2 with: System.out.println(b1.equals((Object)b2));

21. Throwing Exception

public class App {
    public void doRegister(String name, int age) throws UserException, AgeOutOfLimitException {
        if (name.length() < 6) {
            throw new UserException();
        } else if (age >= 60) {
            throw new AgeOutOfLimitException();
        } else {
            System.out.println("User is registered.");
        }
    }

    public static void main(String[] args) throws UserException {
        App t = new App();
        t.doRegister("Matthew", 60);
    }
}

class UserException extends Exception {
}

class AgeOutOfLimitException extends UserException {
}

What is the result? A. User is registered. B. An AgeOutOfLimitException is thrown. C. A UserException is thrown. D. A compilation error occurs in the main method.

  • 由於 line 14 呼叫 App.doRegister() 所傳入之 age 參數為 60，line 5 的 if 條件式會成立，拋出 AgeOutOfLimitException

26. Order Stream (排序串流)

public class Emp {

    String fName;
    String lName;

    public Emp(String fn, String ln) {
        fName = fn;
        lName = ln;
    }

    public String getfName() {
        return fName;
    }

    public String getlName() {
        return lName;
    }
}    

List<Emp> emp = Arrays.asList(
                new Emp("John", "Smith"),
                new Emp("Peter", "Sam"),
                new Emp("Thomas", "Wale"));
        emp.stream()
                // line n1
                .collect(Collectors.toList());

Which code fragment, when inserted at line n1, sorts the employees list in descending order of fName and then ascending order of lName? A. .sorted(Comparator.comparing(Emp::getfName).reversed().thenComparing(Emp::getlName)) B. .sorted(Comparator.comparing(Emp::getfName).thenComparing(Emp::getlName)) C. .map(Emp::getfName).sorted(Comparator.reverseOrder()) D. .map(Emp::getfName).sorted(Comparator.reverseOrder()).map(Emp::getlName).reversed()

  • 選項B，fName 不是遞減排序 (fName，lName 都遞增)
  • 選項C，map會替換掉原本元素，只留下 fName 遞減排序
  • 選項D，Compile Error，第二次 map 方法的元素應該是 String 物件，不是 Emp 物件

77. Default Method of Interface

public interface Moveable<Integer> {

    public default void walk(Integer distance) {
        System.out.println("Walking");
    }

    public void run(Integer distance);
}

Which statement is true? A. Moveable can be used as below:

Moveable<Integer> animal = n -> System.out.println("Running" + n);
animal.run(100);
animal.walk(20);

B. Moveable can be used as below:

Moveable<Integer> animal = n -> n + 10;
animal.run(100);  // run 方法不能有回傳值
animal.walk(20);

C. Moveable can be used as below:

Moveable animal = (Integer n) -> System.out.println(n);
animal.run(100);
Moveable.walk(20);  // walk 方法不能用這樣的方式呼叫

D. Movable cannot be used in a lambda expression. ==Moveable 介面可以使用 Lambda 表示式來實作==

  • run 方法： 需要實作的方法，僅接受傳入一個整數參數，且沒有回傳值
  • walk 方法： Moveable 介面的預設方法，只能在實體化出 Moveable 介面的物件之後才能使用， 無法直接用 Moveable 介面來呼叫
