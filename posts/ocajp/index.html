1. Hello Java

public class HelloJava {

    public static void main(String[] args) {
        StringBuilder message = new StringBuilder("hello java!");
        int pos = 0;
        try {
            for (pos = 0; pos < 12; pos++) {
                switch (message.charAt(pos)) {
                    case 'a':
                    case 'e':
                    case 'o':
                        String uc = Character.toString(message.charAt(pos)).toUpperCase();
                        message.replace(pos, pos + 1, uc);
                }
            }
        } catch (Exception e) {
            System.out.println("Out of limits");
        }
        System.out.println(message);
    }
}

3. Static Variable and Object Variable

public class StaticVariableAndObjectVariable {
    static int i; // 靜態(類別)變數i - 只有一個存在,不會因為類別被實體化就多一個記憶體空間
    int j;  // 物件變數j

    public static void main(String[] args) {
        StaticVariableAndObjectVariable x1 = new StaticVariableAndObjectVariable();
        StaticVariableAndObjectVariable x2 = new StaticVariableAndObjectVariable();
        x1.i = 3;  // == X.i
        x1.j = 4;
        x2.i = 5;  // == X.i
        x2.j = 6;
        System.out.println(
                x1.i + " "
                        + x1.j + " "
                        + x2.i + " "
                        + x2.j);
    }
}

靜態(類別)變數i - 只有一個存在,不會因為類別被實體化就多一個記憶體空間 Console: 5 4 5 6

4. Constructor

public class Sports {

    int num_players;
    String name, ground_condition;

    Sports(int np, String sname, String sground) {
        num_players = np;
        name = sname;
        ground_condition = sground;
    }
}

class Cricket extends Sports {

    int num_umpires;
    int num_substitutes;

    //insert code here

    /*
    Which code fragment can be inserted at line "//insert code here" to enable the code to compile?

    // # B - 
    Cricket() {
        super.ground_condition = "Condition OK";
        super.name = "Cricket";
        super.num_players = 11;
        num_umpires = 3;
        num_substitutes = 2;
    }

    // # C 
    Cricket() {
        this(3,2);
        super(11, "Cricket", "Condition OK");  // must be 1st statement
    }
    Cricket(int nu, int ns) {  // need to add super(.., .., ..);
        this.num_umpires = nu;
        this.num_substitutes = ns;
    }

    // # D 
    Cricket() {
        this.num_umpires = 3;
        this.num_substitutes = 2;
        super(11, "Cricket", "Condition OK");
    }
    */

    // A:
    Cricket() {
        super(11, "Cricket", "Condition OK");
        num_umpires = 3;
        num_substitutes = 2;
    }
}

#B Compiler 會自動在第一行加上super(); but Sports class 沒有 No Arg Constructor #C 即使有加上 super(…, …, …); 放第二行仍然會編譯錯誤，只能放第一行 #D 原因同 #C

8. Polymorphism(多型) and Overriding (覆寫)

public class AlphaBetaGamma {
    static class Alpha {
        public String doStuff(String msg) {
            return msg;
        }
    }

    static class Beta extends Alpha {
        public String doStuff(String msg) {
            return msg.replace('a', 'e');
        }
    }

    static class Gamma extends Beta {
        public String doStuff(String msg) {
            return msg.substring(2);
        }
    }

    public static void main(String[] args) {
        ArrayList<Alpha> strs = new ArrayList<Alpha>();
        strs.add(new Alpha());
        strs.add(new Beta());
        strs.add(new Gamma());
        for (Alpha t : strs) {
            System.out.println(t.doStuff("Java"));
        }
    }
}

What is the result? A. Java 　 Java 　 Java B. Java 　 Jeve 　 va C. Java 　 Jeve 　 ve D. Compilation fails

Alpha, Beta, Gamma 的實體物件都會使用到自己的 doStuff 物件方法

10. Concatenation of Character and String

public class CharToStr {
    public static void main(String[] args) {
        String str1 = "Java";
        char str2[] = {'J', 'a', 'v', 'a'};
        String str3 = null;
        for (char c : str2) {
            str3 = str3 + c;
        }
        if (str1.equals(str3)) {
            System.out.println("Successful");
        } else {
            System.out.println("Unsuccessful");
        }
    }
}

What is the result? A. Successful B. Unsuccessfull C. Compilation fails D. An exception is thrown at runtime

str3 一開始為 null，接著直接使用「+」運算子與字元做字串連接（Java 允許此種作法）， 但是 String str3 = null; 在字串連接時會自動被改為長度為 4 的「null」字串， 輸出結果為「nullJava」，因此結果為 Unsuccessfull。

  • 「+」運算子的運算元必須要至少有一個是字串 String 或者物件 Object，才會被當成字串連結，不然就是加法

            // System.out.println(null + 'c'); // bad operand types (不知型態 + 字元值99)
        String str = null;
        System.out.println(str + "c");                    // nullc
        System.out.println(null + "c");                   // nullc
        System.out.println(str + 'c');                    // nullc
        System.out.println('n' + 'u' + 'l' + 'l' + 'c');  // 542
        System.out.println((String) null + 'c');          // nullc
